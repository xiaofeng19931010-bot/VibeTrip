# Agent原生产品重构方法论与旅行领域落地方案

> 什么是 Vibe Coding？用 AI 当超级助手，告别重流程、重文档，主打**快、轻、有氛围**，先跑通核心体验，再边玩边迭代，把开发从 “搬砖” 变成 “搭乐高”。

## 产品新定位：VibeTrip - 你的氛围感旅行搭子

告别传统 OTA 的工具感，我们要做一个**有温度、有情绪、懂你的旅行搭子**。围绕这个定位，我们主打以下**四大核心功能**与**角色化专属体验**，并以 **MCP Tools 作为能力中心、CLI 作为原生入口、Web Renderer 作为动态界面宿主**，全面拥抱 Agent 原生架构：

1. **一句话规划行程**：告别繁杂攻略。只需一句“下周带爸妈去成都玩3天，预算5000”，AI即可瞬间生成包含机酒、景点、餐饮的完整行程，并智能匹配“长辈舒适节奏”。
2. **自动记录行程**：告别繁琐打卡。旅途中结合LBS地理位置、相册时间戳和碎片化语音，后台静默或半自动为您串联专属旅行轨迹。
3. **自动生成旅行记忆**：告别剪辑焦虑。旅程结束时，AI将行程轨迹、沿途照片与“氛围BGM推荐/播放列表”融合呈现，生成你的专属“氛围感旅行手账”（MVP）或“短视频Vlog”（增强版）。
4. **自动分享旅途**：告别词穷烦恼。一键生成符合小红书/朋友圈调性的“网感文案”与精美长图排版，让分享毫无压力。

**核心体验特点（千人千面 & Agent原生）**：
- **产品入口优先级固定**：`MCP Tools > CLI > Web Renderer`。MCP Tools 是真正的产品能力中心；CLI 是原生入口；Web 仅负责承载 A2UI 动态渲染，不承载传统业务页面。
- **无传统固化 UI (Dynamic GUI) 与双向交互**：告别固定表单和死板卡片，一切基于用户的对话、选择和 MCP Tools 的返回结果动态渲染。采用 **A2UI JSON 协议闭环**设计：`用户操作 → LLM 接收操作并返回 A2UI 标准 JSON → 渲染器解析渲染界面 → 用户继续操作`。
- **渲染器方案极致简洁**：基于 **Shadcn UI + Tailwind CSS** 构建 Web Renderer。只维护少量原子组件（卡片、按钮组、选择器、上传器），通过解析 A2UI JSON 快速装配出动态交互界面。
- **Agent 原生体验 (CLI/MCP)**：
  - 支持**极客视角的 CLI 交互**，在终端输入 `vibetrip plan "成都 3天"` 即可快速获取行程。
  - **基于 MCP (Model Context Protocol) 架构**：VibeTrip 的所有核心能力均封装为标准 MCP 工具；MCP Server 同时支持 `stdio` 与 `SSE/HTTP`。无论是 Cursor、Claude Desktop，还是自定义 Web Renderer，都复用同一套工具与编排引擎。
- **多角色场景自适应**：根据你的出行组合，系统动态生成不同的策略卡片、澄清问题和主题渲染：
  - 👴👵 **子女带父母**：行程节奏自动降速，优先生成大字号卡片、午休提醒、无障碍提示与经典景点建议。
  - 👶🍼 **亲子遛娃**：优先生成母婴设施、互动体验、动物园/乐园等选项卡，并减少高强度切换点。
  - 💑🌹 **情侣度假**：优先生成浪漫主题、氛围感餐厅、小众拍照点与情侣手账模板。
  - 🎒🕶️ **特种兵/闺蜜**：优先生成高密度路线、打卡机位、预算拆分与快节奏提醒。
- **角色选择与纠偏机制**：默认从用户的一句话与历史偏好推断角色；首次进入提供一键选择（带父母/亲子/情侣/闺蜜等），旅途中可随时切换；当系统策略与用户偏好冲突（例如“带父母但想特种兵”），以用户明确指令为准并记录为偏好。
- **告别工具人**：不用切多个页面或多个工具，一句话，搭子帮你搞定所有。
- **情绪价值拉满**：聊天像朋友一样（“宝，我帮你找了不早起的航班哦”），行程配有专属主题色与 BGM 推荐（MVP 不内嵌音乐，避免版权风险）。

## 方案审查（补强版）

### 亮点

- **闭环清晰**：行前“一句话规划”→行中“自动记录”→行后“记忆生成”→“一键分享”，用户价值链条完整。
- **差异化明确**：用“氛围感”+“朋友式交互”解决传统 OTA 工具感。
- **Agent 原生路线正确**：将能力封装为 MCP 工具，天然进入外部 Agent 生态，降低分发成本。

### 主要缺口与风险（建议优先补齐）

- **角色化的可执行策略未落地**：目前是体验描述，缺少“策略参数化”与“约束优先级”。建议把角色适配落实到可配置的策略项（节奏、步行强度、起床时间、午休、预算分配、无障碍、亲子设施、浪漫点位、拍照机位等），并定义冲突时的优先级。
- **自动记录的隐私/合规与电量成本**：LBS、相册与语音采集会触发高敏权限与耗电问题。建议明确默认策略：首次授权说明、按需采集、低功耗定位、端侧预处理、随时暂停、数据可导出/可删除。
- **“旅行记忆=视频”风险偏大**：音视频合成、模板渲染、版权音乐、端侧性能在 6 周内容易失控。建议 MVP 先交付“图文手账/长图海报/Markdown 旅行记”，视频作为增强版分阶段上线。
- **MCP 对外开放的安全面与 BYOK**：需要定义认证、配额、审计与工具权限边界；针对 LLM 必须支持 BYOK (Bring Your Own Key) 并保证用户的 Key 在流转中的绝对安全（不落库或端到端加密），避免被滥用。

### 建议补齐的最小规格（让计划可交付）

#### 统一数据模型（全端/CLI/MCP一致）

- `Trip`：目的地、日期、人群角色、预算、偏好、状态
- `Itinerary`：按天分组的 `ItineraryItem`（交通/住宿/景点/餐饮/休息）
- `Capture`：位置点、照片引用、语音转写、手动笔记
- `MemoryArtifact`：旅行记忆（手账/长图/视频）的版本与素材引用
- `SharePackage`：平台适配（朋友圈/小红书）文案与素材清单

#### MCP Server 最小工具集（建议从 8 个工具起步）

- `plan_trip`：一句话→结构化行程（支持角色参数）
- `revise_itinerary`：按指令修改（“别早起”“加午休”“多亲子”）
- `persist_trip`：保存/更新行程与状态
- `start_capture` / `stop_capture`：开始/暂停自动记录
- `ingest_media`：导入照片/语音/笔记
- `generate_memory`：生成旅行手账/长图（MVP）
- `generate_share`：生成社媒文案+素材清单

#### CLI 最小命令（与 MCP 一一对应）

- `vibetrip plan "…" --role parents|family|couple --days 3 --budget 5000`
- `vibetrip trip save|show|export`
- `vibetrip capture start|stop|status`
- `vibetrip memory generate --format handbook|poster`
- `vibetrip share generate --channel xhs|moments`

以上补强点补齐后，计划会从“产品故事”升级为“可落地规格”，同时能保证 MCP/CLI/GUI 三端复用同一套能力与数据流，交付风险显著降低。
